# كيفية تشغيل المشروع ومصادر الأخطاء الشائعة

## تشغيل الواجهة (Client)

من **جذر المشروع**:
```bash
npm start
```
أو من مجلد العميل:
```bash
cd client
npm start
```
يفتح التطبيق على: http://localhost:3000

---

## تشغيل السيرفر (Backend)

من **جذر المشروع**:
```bash
npm run start:server
```
أو يدوياً:
```bash
cd server
npm run build
npm start
```
**مهم:** السيرفر يعتمد على مجلد `dist`، لذلك يجب تنفيذ `npm run build` مرة واحدة قبل أول تشغيل.

لتشغيل السيرفر بوضع التطوير (مع إعادة التشغيل التلقائي):
```bash
npm run dev:server
```

## مصدر الأخطاء الشائعة

| الرسالة | السبب | الحل |
|--------|--------|------|
| **Missing script: "start"** | تشغيل `npm start` من الجذر قبل إضافة السكربت | استخدم `npm start` من الجذر (الآن يعمل)، أو `cd client && npm start` |
| **Cannot find module '...dist/index.js'** | تشغيل السيرفر بدون بناء | من مجلد `server`: نفّذ `npm run build` ثم `npm start` |
| **The string is missing the terminator** (في PowerShell) | مسار المشروع يحتوي على أحرف عربية أو مسافات | افتح الـ Terminal من داخل Cursor أو استخدم مسار قصير؛ أو شغّل من مجلد `client` مباشرة: `cd client` ثم `npm start` |
| **EADDRINUSE / Port 3000 already in use** | المنفذ 3000 مستخدم | أغلق التطبيق الآخر الذي يستخدم المنفذ، أو غيّر المنفذ في `client/.env` أو أوقف العملية من إدارة المهام |
| **bash: command not found** | استخدام أمر Linux (مثل `bash`) في Windows | استخدم أوامر PowerShell أو CMD فقط؛ لا تستخدم `bash` أو `curl ... \| bash` في Windows |

## ترتيب التشغيل الموصى به

1. **تثبيت الحزم (مرة واحدة):**
   ```bash
   npm run install:all
   ```

2. **تشغيل السيرفر:**
   ```bash
   npm run start:server
   ```
   أو للتطوير: `npm run dev:server`

3. **في نافذة Terminal أخرى – تشغيل الواجهة:**
   ```bash
   npm start
   ```

إذا ظهر خطأ غير مذكور هنا، انسخ رسالة الخطأ كاملة من الـ Terminal وشاركها لمعرفة المصدر بدقة.
